                                                          I'm still not 100% sure what's exactly right, but I do believe it should be dot(N, V)>. I'll explain my reasoning below and I'm curious what you think:

                                                          - Officially the Fresnel is based on the angle between N and V as you can see here (if you can count Wiki as 'official', but I've seen it mentioned like this on multiple occasions).
                                                          - By using the halfway vector instead of the view vector the Fresnel value is still based on view direction. I'll have to try it later (at family for Christmas now); I believe the increase in specularity will slightly differ, but it will still be way too strong at rougher surfaces.
                                                          - Not taking roughness into account doesn't make much sense in the microfacet model; as the roughness (thus the microfacet alignment at microlevel) should influence the level of specular reflection. I didn't look at Lagarde's direct PBR implementations, but in prefiltered environment maps most implementations use the default Fresnel-Schlick function (without roughness) as the halfway vector input there is generated based on roughness which implicitly introduces a roughness term in the Fresnel equation. We don't have this with direct lighting as the V/H vector is directly based on the (always smooth) surface which is why we fake some per-fragment roughness by directly including it in the Fresnel equation.

                                                          Let me know if I made some incorrect assumptions or statements as I don't consider myself 100% a PBR expert :)

                                                              Josh Edmondson JoeyDeVries • 3 years ago

                                                              Notice in the last paragraph of the Fresnel wiki article you linked, it says ‘the normal in the above computation is replaced by the halfway vector’, the halfway vector is also used in equation 2 of ‘Real Shading in Unreal Engine 4’.

                                                              To give a brief overview of the microsurface reflectance theory and why we use the halfway vector;
                                                              Let’s start with the most simple case, a completely smooth, mirror like surface. A ray of light will reflect off the surface at the same angle it arrived (angle of incidence = angle of reflection). This means the normal is always halfway between the light vector and the reflection vector. So in order to view any reflected light, the view vector must be equal to the reflected vector (because the surface is perfectly smooth). When V equals R, H is equal to N. This mean when testing to see if V receives any reflected light we could check if V equals R or we could check if H equals N (both give the same result). Then we apply the Fresnel attenuation. F(N, V). Done. Reflection off a smooth surface calculated.

                                                              But what if our surface is rough?... Like the macrosurface.
                                                              Microsurface theory makes the assumption that our rough surface is made up many perfectly smooth microsurfaces each with their own normal. First we ask how much of the microsurface area has its normal exactly aligned with H (because, we can only see the light if the normal of the microsurface is aligned H). This is what the normal distribution function does. NDF(H, N, Roughness). This means all remaining microsurfaces have a normals of H. Next we ask how much the microsurface area is self shadowing or occluded in the viewing direction. That’s what the geometry function does. G(V, L, N, H, Roughness). Finally we ask how much the Fresnel attenuation affects the reflection off the microsurface. You are not wrong in saying ‘Officially the Fresnel is based on the angle between N and V’, but remember we are testing with the microsurface normal, which happen to be equal to the H, otherwise it would not have passed the NDF test. So when we pass H to the Fresnel function we are just passing the microsurface normal, not the macrosurface normal. F(V, H). Multiply those three things together and you’re done. The ‘4 * dot(N, V) * dot (N, L)’ in the denominator is just part of ‘transforming incident irradiance onto the microsurface and then transform the scattered radiance back to the macrosurface’(EGSR07-btdf) and some H==N integral math magic.

                                                              Regarding use of a roughness value in the Fresnel equation, when we multiply by NDF and G, all roughness properties have already been considered. shadowing, occlusion, Normal distribution, are already accounted for. But for Indirect lighting that is a different story.

                                                              Many techniques for prefiltering specular environment maps include approximations. A common one being N equals V. Quoting Lagarde’s ‘Adopting a physically based shading model’ article, this causes the edges to become two bright ‘since there you are averaging incoming light colors from many directions, but using a single Fresnel value computed for the reflection direction.’ This is why he added the empirical ‘fudge factor’ when calculating Fresnel for prefiltered environment maps.

                                                              I hope I helped explain things and didn’t make the topic more confusing than it already is :D
                                                              If you’re looking for more good papers to read on the PBR let me know, I’ve got a big list of them.

                                                                                                                                                    I found a bug in the code.

                                                                                                                                                    I would get these weird random black spots on my mesh. After debugging i realized the mistake is in the fresnelSchlick function. You never checks if cosTheta is bigger then 1. Mathematically, this is not possible, but since we are working with floats, sometimes you can get the cosine to be slightly bigger then 1. When that happens, in the pow function( it tries to calculate (1-cosTheta)^5), it fails, since the function cant calculate a negative number to a power, and it returns NaN, and it discards the pixel. So in the fresnelSchlick function you need to check if cosTheta is bigger then 1.0, and if it is, assign it to 1.0.

                                                                                                                                                    vec3 fresnelSchlick(float cosTheta, vec3 F0)
                                                                                                                                                    {
                                                                                                                                                    if(cosTheta>1.0)
                                                                                                                                                    cosTheta=1.0;
                                                                                                                                                    float p=pow(1.0 - cosTheta, 5.0);
                                                                                                                                                    return F0 + (1.0 - F0) * p;
                                                                                                                                                    }
